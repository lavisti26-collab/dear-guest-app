PART 2 — TOP 5 CRITICAL ISSUES
--------------------------------------------------------------------------------
1. ALTERNATIVE LAYOUTS ARE SKELETON PLACEHOLDERS (Issue #1)
   - Reason: Selecting layouts like "Apple Product", "Cinematic", or "Khmer Traditional" completely breaks the public-facing invitation, silently deleting critical sections (timeline, bank info, map, guest wishes). This invalidates the core product promise of custom templates.

2. RSVP GUEST NAME COLLISION OVERWRITES DATA (Issue #6)
   - Reason: Updating RSVPs by exact name matching instead of unique IDs will overwrite data if multiple guests share a common name (extremely common in Cambodia). One guest submitting an RSVP could overwrite another guest's attendance status.

3. MISSING NOTIFICATION AND ACTIVITY LOG TABLES (Issue #3)
   - Reason: The database schema is missing the `notifications` and `super_admin_activity_log` tables entirely. Any admin attempt to broadcast notifications or review super admin logs crashes or yields database errors.

4. SEARCH ENTER COLLISION TRASHES GUEST WISHES (Issue #5)
   - Reason: Because the wishes search bar is placed inside the wish creation `<form>`, typing a search query and pressing "Enter" will submit a partial or blank wish, triggering validation errors and locking the UI.

5. KHMER SCRIPT OVERFLOW AND TEXT CLIPPING (Issue #7)
   - Reason: Since Khmer script does not use spaces, browsers do not know where to wrap sentences. Without `word-break: break-all` or `overflow-wrap: break-word`, the Khmer invitation details, names, and messages clip or overflow on mobile screens.

--------------------------------------------------------------------------------
PART 3 — QUICK WINS (Low Effort, High Impact)
--------------------------------------------------------------------------------
1. Move the wishes search input outside the `<form>` tag in `WishesSection.tsx` (takes 1 minute, fixes the Enter form submission bug).
2. Use the guest's unique ID (`guest.id`) instead of guest name in `AdminDashboard.tsx` `downloadQR()` for the DOM SVG element selector (prevents failure for Khmer script names).
3. Hide the "Super Admin" option from the "Role Elevator" for regular admin accounts to prevent frustrating Access Denied redirection loops.
4. Remove the `supabase.auth.signOut()` trigger immediately after signup in `AdminLogin.tsx` so users stay logged in and see the pending approval screen directly.
5. Add `word-break: break-word` and `overflow-wrap: break-word` rules to primary text wrapper CSS classes in `index.css` to fix Khmer script mobile clipping.
